## Task 3: Harden signal and edge-case handling in binary forwarding

**Parallel:** yes (batch 2) | **Blocked by:** Task 2 | **Owned files:** `npm/specgate/bin/specgate.js`

### Context

The wrapper's signal handling maps signal names to numeric codes, but several edge cases need hardening:
1. The `SIGSTKFLT` signal does not exist on macOS -- will throw if received
2. The unreachable code path at the bottom of the `for` loop (after the `result.error` check) logs a generic message that is confusing
3. No test coverage for the case where `spawnSync` returns `{ signal: 'SIGTERM' }`

### Step 1: Write failing tests for signal handling

In `npm/specgate/bin/specgate.test.js`:

```javascript
describe("signal handling edge cases", () => {
  it("handles unknown signals gracefully", () => {
    // Create a fake binary that self-signals with SIGTERM
    const tmpDir = fs.mkdtempSync(path.join(os.tmpdir(), "specgate-signal-"));
    const fakeBin = path.join(tmpDir, "fake-specgate");
    fs.writeFileSync(fakeBin, `#!/usr/bin/env node
process.kill(process.pid, "SIGTERM");
`);
    fs.chmodSync(fakeBin, 0o755);

    try {
      const result = runWrapperWithBlockedTypescript(["check"], {
        SPECGATE_NATIVE_BIN: fakeBin,
      });
      // SIGTERM = 15, exit code should be 128 + 15 = 143
      assert.strictEqual(result.status, 143, `expected exit 143 (128+SIGTERM), got ${result.status}`);
      assert.match(result.stderr, /killed by signal/i);
    } finally {
      fs.rmSync(tmpDir, { recursive: true, force: true });
    }
  });
});
```

**Step 3: Remove platform-specific signals that cause errors on macOS**

```javascript
function signalExitCode(signalName) {
  if (process.platform === "win32") {

}
```

Replace the inline signal code calculation in `runNativeSpecgate()` with a call to `signalExitCode(result.signal)`.

## Task 4: Platform support matrix validation tests

**Parallel:** yes (batch 2) | **Blocked by:** Task 2 | **Owned files:** `npm/specgate/bin/specgate.test.js`

### Context

The existing tests already cover this partially, but we need to add Windows support (for future-proofing) and ensure the full support matrix is exercised.

### Step 1: Add support matrix contract test

In `npm/specgate/bin/specgate.test.js`:

```javascript
describe("support matrix contract", () => {
  const SUPPORT_MATRIX = [
    { platform: "linux", arch: "x64", rustTarget: "x86_64-unknown-linux-gnu" },
    { platform: "darwin", arch: "x64", rustTarget: "x86_64-apple-darwin" },
    { platform: "darwin", arch: "arm64", rustTarget: "aarch64-apple-darwin" },
  ];

  for (const { platform, arch, rustTarget } of SUPPORT_MATRIX) {
    it(`maps ${platform}/${arch} to native/${platform}/${arch}/specgate`, () => {
      const result = runWithPlatform(platform, arch, `
        const path = require("node:path");
        const binDir = path.dirname(wrapperPath);
        const binaryName = process.platform === "win32" ? "specgate.exe" : "specgate";
        const primary = path.resolve(binDir, "..", "native", process.platform, process.arch, binaryName);
        const exists = require("node:fs").existsSync(primary);
        process.stdout.write(JSON.stringify({ primary, exists }));
      `);
      assert.strictEqual(result.status, 0, result.stderr);
      const data = JSON.parse(result.stdout);
      assert.ok(
        data.primary.includes(`native/${platform}/${arch}/`),
        `primary candidate should include native/${platform}/${arch}/, got ${data.primary}`
      );
    });
  }

  it("release matrix targets match wrapper platform map", () => {
    // The 3 release targets must each have a corresponding
    // platform/arch combination in the wrapper's nativeCandidates()
    const releaseTargets = [
      "x86_64-unknown-linux-gnu",
      "x86_64-apple-darwin",
      "aarch64-apple-darwin",
    ];

    const wrapperPlatformMap = {
      "x86_64-unknown-linux-gnu": { platform: "linux", arch: "x64" },
      "x86_64-apple-darwin": { platform: "darwin", arch: "x64" },
      "aarch64-apple-darwin": { platform: "darwin", arch: "arm64" },
    };

    for (const target of releaseTargets) {
      assert.ok(
        wrapperPlatformMap[target],
        `release target ${target} must have wrapper platform mapping`
      );
    }
  });
```

```rust
fn copy_dir_recursive_excluding(from: &Path, to: &Path, exclude: &[&str]) {
    for entry in walkdir::WalkDir::new(from)
        .follow_links(false)
        .into_iter()
        .filter_map(std::result::Result::ok)
    {
        let src_path = entry.path();
        let rel = src_path.strip_prefix(from).expect("strip fixture prefix");
        let rel_str = rel.to_str().expect("utf8");
        let normalized = rel_str.replace('\\', "/");

        if exclude.iter().any(|exc| normalized.ends_with(exc)) {
            continue;
        }

        let dst_path = to.join(rel);

        if entry.file_type().is_dir() {
            fs::create_dir_all(&dst_path).expect("mkdir fixture copy dir");
            continue;
        }

        if let Some(parent) = dst_path.parent() {
            fs::create_dir_all(parent).expect("mkdir fixture copy parent");
        }
        fs::copy(src_path, &dst_path).expect("copy fixture file");
    }
}
```
